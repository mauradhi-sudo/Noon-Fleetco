                    GOOGLE CLOUD DEPLOYMENT (20 MINUTES)

═══════════════════════════════════════════════════════════════════════════════

STEP 1: SETUP GOOGLE CLOUD PROJECT (5 minutes)
────────────────────────────────────────────────────────────────────────────

1. Go to: https://console.cloud.google.com
2. Click "Select a Project" → "NEW PROJECT"
3. Project name: noon-fleet-hrms
4. Click "CREATE"
5. Wait for project creation
6. Select the new project


STEP 2: ENABLE REQUIRED APIs (3 minutes)
────────────────────────────────────────────────────────────────────────────

1. In Google Cloud Console
2. Go to "APIs & Services" → "Library"
3. Search for and ENABLE these APIs:
   ✅ Cloud Run API
   ✅ Cloud SQL Admin API
   ✅ Cloud Build API
   ✅ Artifact Registry API

4. Wait 1-2 minutes for APIs to enable


STEP 3: CREATE DATABASE (Cloud SQL) (5 minutes)
────────────────────────────────────────────────────────────────────────────

1. Go to "Cloud SQL" in left menu
2. Click "CREATE INSTANCE"
3. Choose "PostgreSQL"
4. Instance ID: noon-hrms-db
5. Password: [Set strong password]
6. Database version: PostgreSQL 15
7. Region: asia-southeast1 (or closest to you)
8. Machine type: Shared Core / 1 vCPU, 0.614 GB
9. Click "CREATE INSTANCE"
10. Wait 5-10 minutes for database creation

After creation:
11. Click on instance → "CONNECTIONS" tab
12. Click "Private IP" and "Setup connection"
13. Enable "Private IP"
14. Wait for network setup
15. Copy the PRIVATE IP address


STEP 4: CREATE DATABASE & TABLES (2 minutes)
────────────────────────────────────────────────────────────────────────────

1. Click on instance → "DATABASES" tab
2. Click "CREATE DATABASE"
3. Database name: noon_hrms
4. Click "CREATE"

Now add tables:
5. Click "DATABASES" → "noon_hrms" → "CONNECT"
6. Click "Cloud Shell" at bottom
7. Copy entire contents of DATABASE-MIGRATIONS.sql
8. Paste in Cloud Shell and execute
9. Wait for success ✅


STEP 5: CREATE CLOUD RUN SERVICE (5 minutes)
────────────────────────────────────────────────────────────────────────────

1. Go to "Cloud Run" in left menu
2. Click "CREATE SERVICE"
3. Container image URL: gcr.io/cloudrun/hello (we'll update this)
4. Service name: noon-fleet-hrms
5. Region: asia-southeast1
6. Authentication: Allow unauthenticated invocations
7. Click "CREATE"
8. Wait for service creation


STEP 6: DEPLOY CODE (Docker Build) (5 minutes)
────────────────────────────────────────────────────────────────────────────

1. Create Dockerfile in your project root:

---START DOCKERFILE---
FROM node:18-alpine

WORKDIR /app

COPY package*.json ./
RUN npm install --production

COPY . .

EXPOSE 8080
ENV PORT=8080

CMD ["npm", "start"]
---END DOCKERFILE---

2. Push code to GitHub (if not already)
3. Go to "Cloud Build" in console
4. Click "CREATE"
5. Connect GitHub repository
6. Branch: main
7. Build configuration: Dockerfile
8. Runtime settings: Set environment variables
9. Click "CREATE AND RUN"
10. Wait for build to complete


STEP 7: SET ENVIRONMENT VARIABLES (2 minutes)
────────────────────────────────────────────────────────────────────────────

In Cloud Run Service:
1. Click on service → "EDIT & DEPLOY NEW REVISION"
2. Go to "Runtime settings" → "Set environment variables"
3. Add:

   PORT=8080
   DATABASE_URL=postgresql://postgres:PASSWORD@PRIVATE_IP:5432/noon_hrms
   JWT_SECRET=your-super-secret-key-12345
   SUPER_ADMIN_EMAIL=mauradhi@noon.com
   NODE_ENV=production

Replace:
   - PASSWORD: Your Cloud SQL password
   - PRIVATE_IP: Cloud SQL private IP from Step 3

4. Click "DEPLOY"
5. Wait for deployment


STEP 8: CONFIGURE NETWORK (3 minutes)
────────────────────────────────────────────────────────────────────────────

1. Go to "VPC networks" → "Serverless VPC Connector"
2. Create connector:
   Name: noon-hrms-connector
   Region: asia-southeast1
   Network: default
   Subnet IP range: 10.8.0.0/28
3. Click "CREATE"
4. Wait for connector creation

5. Go back to Cloud Run service
6. Click "EDIT & DEPLOY NEW REVISION"
7. Go to "VPC Connector" section
8. Select: noon-hrms-connector
9. Click "DEPLOY"


STEP 9: TEST DEPLOYMENT (2 minutes)
────────────────────────────────────────────────────────────────────────────

1. Get your Cloud Run service URL (shown in console)
2. Test health endpoint:
   https://YOUR-SERVICE-URL.run.app/api/health
   Should return: {"ok": true, "status": "HRMS running"}

3. Test admin portal:
   https://YOUR-SERVICE-URL.run.app/admin-portal.html

4. Test employee portal:
   https://YOUR-SERVICE-URL.run.app/employee-portal.html

                              TROUBLESHOOTING

Database Connection Issues:
1. Check VPC Connector is created
2. Verify Cloud SQL instance is running
3. Check DATABASE_URL format
4. Enable Cloud SQL Client role for Cloud Run

Service Not Starting:
1. Check Cloud Build logs
2. Verify Dockerfile is correct
3. Check environment variables set
4. Review Cloud Run logs

API Not Responding:
1. Check Cloud Run logs (Logs section)
2. Test health endpoint: /api/health
3. Verify database is running
4. Check network connectivity

Portal Not Loading:
1. Clear browser cache
2. Check JavaScript console (F12)
3. Verify API endpoints are correct
4. Check CORS settings if needed
