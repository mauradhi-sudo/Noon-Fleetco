                           DEPLOY IN 5 STEPS
================================================================================

STEP 1: SETUP GITHUB (2 minutes)
────────────────────────────────────────────────────────────────────────────

1. Open GitHub
2. Create new repository: noon-fleet-hrms
3. Clone locally:
   git clone https://github.com/YOUR-USERNAME/noon-fleet-hrms.git
   cd noon-fleet-hrms

4. Copy all files from NOON-Fleetco folder to this repo
5. Push to GitHub:
   git add .
   git commit -m "Initial NOON HRMS - Complete unified system"
   git push -u origin main


STEP 2: SETUP RENDER (2 minutes)
────────────────────────────────────────────────────────────────────────────

1. Go to https://render.com
2. Login/Sign up
3. Click "New Web Service"
4. Connect GitHub repository
5. Configuration:
   Name: noon-fleet-hrms
   Root Directory: /
   Build Command: npm install
   Start Command: npm start
   Plan: Free or Paid

6. Click "Create Web Service"


STEP 3: SETUP DATABASE (3 minutes)
────────────────────────────────────────────────────────────────────────────

1. In Render Dashboard, click "New PostgreSQL"
2. Configuration:
   Name: noon-hrms-db
   Region: Singapore (or closest to you)
   Plan: Free or Paid
3. Click "Create Database"
4. Wait for database to be created
5. Copy DATABASE_URL from database page
6. Keep this URL - you'll need it in Step 4


STEP 4: CONFIGURE ENVIRONMENT (1 minute)
────────────────────────────────────────────────────────────────────────────

1. Go back to Web Service (noon-fleet-hrms)
2. Click "Environment" tab
3. Add these variables:
   PORT = 10000
   DATABASE_URL = [Paste the PostgreSQL URL from Step 3]
   JWT_SECRET = [Generate random key or use: your-secret-key-12345]
   SUPER_ADMIN_EMAIL = mauradhi@noon.com

4. Click "Save"
5. Web Service will auto-redeploy


STEP 5: RUN DATABASE MIGRATIONS (2 minutes)
────────────────────────────────────────────────────────────────────────────

1. Go to PostgreSQL database in Render
2. Click "Connect" → "Browser"
3. Open the SQL editor
4. Copy entire contents of DATABASE-MIGRATIONS.sql
5. Paste into the SQL editor
6. Click "Execute"
7. Wait for success message ✅
8. You should see all tables created


STEP 6: VERIFY DEPLOYMENT (2 minutes)
────────────────────────────────────────────────────────────────────────────

1. Go to Render Web Service
2. Watch the deployment logs - should say "Build successful"
3. Get your app URL (e.g., https://noon-fleet-hrms-xxxxx.onrender.com)
4. Test in browser:
   https://noon-fleet-hrms-xxxxx.onrender.com/api/health
   Should return: {"ok": true, "status": "HRMS running"}

================================================================================
                              TROUBLESHOOTING
================================================================================

If something doesn't work:

1. Check Render Logs
   → Go to Web Service → Logs tab
   → Should say "HRMS running on port 10000"

2. Test API Health
   → curl https://your-app.onrender.com/api/health
   → Should return JSON with "ok": true

3. Verify Database
   → Go to PostgreSQL → Connect → Browser
   → Run: SELECT * FROM employees;
   → Should show tables exist

4. Clear Browser Cache
   → Press Ctrl+Shift+Delete
   → Select "All time"
   → Clear everything
   → Try again

5. Check Network Tab
   → Open portal
   → Press F12 → Network tab
   → See if API calls return 200 or error
   → Check Response tab for error details
